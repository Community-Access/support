# Support Playbook

This playbook defines how maintainers triage and resolve support requests in this repository.

## Triage Rhythm

- Active cohort periods: review issues and discussions daily.
- Off-cycle: review at least three times per week.

## First Response Standard

Within 24 to 48 hours, every new issue should receive:

1. Acknowledgement of the question.
2. One concrete next step.
3. A request for missing details only if needed.
4. Label updates reflecting issue type.

## Triage Decision Tree

1. Determine scope:
   - Support and troubleshooting: handle in this repo.
   - Curriculum change request: route to main workshop repo.
   - Template automation change request: route to learning-room-template.
2. Add labels.
3. Respond with a next action.
4. Follow up until resolved or escalated.

## Escalation Rules

Use `escalated` when:

- A learner is blocked for more than 48 hours.
- The issue may affect multiple learners.
- The issue might be an automation regression.

## Privacy Rules

- Do not ask users to post secrets, tokens, or private links.
- Move sensitive account details to private facilitator contact.

## Resolution Criteria

Mark `resolved` when:

- User confirms the fix works, or
- Maintainer confirms actionable resolution and no further blocker remains.
