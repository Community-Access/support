# Contributing to Community Access Support

Use this repository for support operations, troubleshooting requests, and community Q&A across the workshop ecosystem.

## Routing Guidance

- Support requests and tracked blockers: https://github.com/Community-Access/support/issues
- Q&A and async mentoring: https://github.com/Community-Access/support/discussions
- Workshop curriculum source changes: https://github.com/Community-Access/git-going-with-github
- Learning Room template source changes: https://github.com/Community-Access/learning-room-template
- Accessibility Agents source changes: https://github.com/Community-Access/accessibility-agents

## Issue Expectations

When opening an issue, include:

1. What you were trying to do.
2. What you expected to happen.
3. What actually happened.
4. Steps to reproduce.
5. Platform and accessibility tooling details when relevant.

## Community Standards

- Be respectful and constructive.
- Do not post secrets, private links, or personal data.
- Use clear titles and include enough context for maintainers to triage quickly.
